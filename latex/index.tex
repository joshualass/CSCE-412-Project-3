This project simulates a dynamic load-\/balancing network that routes incoming web requests to a cluster of web servers. It is designed to run for a specified number of clock cycles (e.\+g., 10,000) and dynamically scale the number of active servers based on the current queue size to optimize network performance.

\begin{DoxyItemize}
\item {\bfseries{Dynamic Auto-\/\+Scaling:\+}} Servers are automatically activated or deactivated depending on traffic volume and queue depth.
\item {\bfseries{IP Firewall:\+}} Malicious requests falling within a user-\/defined IP range are actively blocked and dropped before reaching the servers.
\item {\bfseries{Variable Task Times:\+}} Processing times for incoming requests are randomized within a dynamic, user-\/provided range.
\item {\bfseries{Comprehensive Metrics:\+}} Calculates and logs average queue wait times, total turnaround times, and server utilization histograms.
