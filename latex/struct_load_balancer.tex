\doxysubsection{Detailed Description}
Manages a queue of requests and scales Webservers dynamically based on load. 


\begin{DoxyItemize}
\item The \doxylink{struct_load_balancer}{Load\+Balancer} acts as the middleman between the \doxylink{struct_switch}{Switch} and the individual Webservers. It contains a built-\/in firewall to drop malicious IPs, an internal queue to hold pending requests, and logic to activate or deactivate servers based on current traffic demands. 
\end{DoxyItemize}

Simulates a single clock cycle for the \doxylink{struct_load_balancer}{Load\+Balancer} and its servers. 

Evaluates scaling thresholds, delegates requests, and advances time.


\begin{DoxyItemize}
\item Evaluates auto-\/scaling thresholds, assigns queued requests to idle active servers, and advances the clock cycle for all managed Webservers.
\item Every clock cycle, the load balancer checks if the cooldown timer (check\+\_\+countdown) has reached zero. If so, it evaluates the ratio of queued requests to active servers. If traffic is too high, it activates an offline server. If traffic is low, it deactivates an online server to save resources. Finally, it delegates requests from the front of the queue to available active servers and calls \doxylink{struct_load_balancer_ae1d0ad0027bfb13e39b1038ffc4ec28d}{simulate\+Clock\+Cycle()} on every server. 
\end{DoxyItemize}
