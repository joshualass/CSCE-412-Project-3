Load Balancer Simulation Project

Introduction

This project simulates a dynamic load-balancing network that routes incoming web requests to a cluster of web servers. It is designed to run for a specified number of clock cycles (e.g., 10,000) and dynamically scale the number of active servers based on the current queue size to optimize network performance.

Key Features

  • Dynamic Auto-Scaling: Servers are automatically activated or deactivated depending on traffic volume and queue depth.
  • IP Firewall: Malicious requests falling within a user-defined IP range are actively blocked and dropped before reaching the servers.
  • Variable Task Times: Processing times for incoming requests are randomized within a dynamic, user-provided range.
  • Comprehensive Metrics: Calculates and logs average queue wait times, total turnaround times, and server utilization histograms.

How to Run

The simulation is executed via the command line with the following parameters: make ./loadbalancer (total_cycles) (num_requests) (check_interval) (firewall_low) (firewall_high) (min_task_time) (max_task_time)
