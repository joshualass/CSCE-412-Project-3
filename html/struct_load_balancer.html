Detailed Description

Manages a queue of requests and scales Webservers dynamically based on load.

  • The LoadBalancer acts as the middleman between the Switch and the individual Webservers. It contains a built-in firewall to drop malicious IPs, an internal queue to hold pending requests, and logic to activate or deactivate servers based on current traffic demands.

◆ simulateClockCycle()

void LoadBalancer::simulateClockCycle ( )

Simulates a single clock cycle for the LoadBalancer and its servers.

Evaluates scaling thresholds, delegates requests, and advances time.

  • Evaluates auto-scaling thresholds, assigns queued requests to idle active servers, and advances the clock cycle for all managed Webservers.
  • Every clock cycle, the load balancer checks if the cooldown timer (check_countdown) has reached zero. If so, it evaluates the ratio of queued requests to active servers. If traffic is too high, it activates an offline server. If traffic is low, it deactivates an online server to save resources. Finally, it delegates requests from the front of the queue to available active servers and calls simulateClockCycle() on every server.
